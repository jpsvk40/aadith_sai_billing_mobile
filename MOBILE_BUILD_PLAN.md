## PHASE 0 — Backend: R2 object storage for scanned images (do first)
**Why:** `scannedInvoiceUrl` currently stores the whole image as **inline base64 in the DB row** (~1.6 MB each; ~418 MB total on prod). This caused the 64 MB Neon / OOM problem we already fixed in the reports. Mobile capture makes it worse. This is a **shared backend change that benefits web too** (web just gets a URL instead of a data URL).

Tasks:
1. **R2 bucket (private)** + upload helper in backend. On scan/upload: push bytes to R2, store only the key/URL in `scannedInvoiceUrl` (and the new credit-note `scannedCreditNoteUrl`).
2. **Serve securely (financial docs):** either
   - (a) **short-lived presigned GET URLs** generated by backend on demand *(recommended)*, or
   - (b) auth proxy `GET /api/.../scanned-image/:id` that checks JWT and streams from R2.
   Do **not** make the bucket public.
3. **Dual-mode during migration:** image-serving reads "if value starts with `data:` → serve inline (legacy); else treat as R2 key." Nothing breaks mid-backfill.
4. **One-time backfill script:** read each base64 `scannedInvoiceUrl` → upload to R2 → replace column with key. Run in background.
5. **Web UI impact = tiny:** `<img src={...}>` / "View Scanned Bill" renders an https R2 URL same as a data URL — plus removes base64 bloat from every vendor-purchase query.
6. Decide provider: **Cloudflare R2** (chosen direction) vs S3/Supabase. Configure CORS for browser/mobile if serving directly.

Also (infra, not strictly R2): add `connect_timeout=30` / keep-warm so the app's first call after Neon idle doesn't `P1001`.

## PHASE 3 — AI Scanner (Admin) — standalone + embedded
- **Models** `data/models/`: `scanned_invoice_model.dart`, `scanned_credit_note_model.dart` (mirror `{extracted, candidates, matchedPurchase}`).
- **Repository** `data/repositories/ai_scan_repository.dart`: `scanVendorBill(filePath)`, `scanCreditNote(filePath)` via `client.uploadFile(...)`.
- **Screens** `features/scanner/screens/scan_screen.dart`: Camera/Gallery (image_picker) → upload → loading → **prefilled review form** → "Create Purchase" / "Create Credit Note".
- **Port the two web safeguards to Dart:** (1) invoice **date-year sanity** (flag >15 months old / future / decade-off — GPT misreads 2026→2016/2023), (2) **fuzzy mismatch warning** (`_looseMatch` + Levenshtein) so a scanned doc whose invoice/vendor/GSTIN doesn't match can't attach silently.
- **Router:** `/scan` standalone; also launched from `add_credit_note_screen` and a "New Purchase from scan" entry.

## Lessons carried from the web work (apply on mobile)
- **Never send blob columns to a list endpoint** (caused OOM). Mobile lists must be lean + paginated.
- **Migrations apply on Render deploy**, but cold-start `P1001` can block the deploy — warm Neon / retry / add connect_timeout.
- AI extracts **amounts** reliably but **identifiers (invoice no., GSTIN, year)** can be OCR-noisy → keep the date-year + mismatch guards.
